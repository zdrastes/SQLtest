Схема базы данных

Таблицы связаны между собой через ключи: employees.id → employee_projects.employee_id, projects.id → employee_projects.project_id, departments.id → employees.department_id

employees

Таблица сотрудников

id INT PRIMARY KEY
full_name VARCHAR(100)
department_id INT
position VARCHAR(100)
salary DECIMAL(10,2)

projects

Таблица проектов

id INT PRIMARY KEY
name VARCHAR(100)
client VARCHAR(100)
budget DECIMAL(12,2)
status VARCHAR(20)

employee_projects

Таблица назначений сотрудников на проекты

employee_id INT
project_id INT
role VARCHAR(50)
hours_worked INT

departments

Таблица отделов

id INT PRIMARY KEY
name VARCHAR(100)
manager_id INT

Связи: employees.department_id → departments.id | employee_projects.employee_id → employees.id | employee_projects.project_id → projects.id

SQL Запросы к базе данных

1 Вывести количество сотрудников в каждом отделе. Столбцы «отдел», «количество сотрудников».
2 Вывести всех сотрудников отделов «Отдел продаж» и «IT отдел». Вывести колонки «ФИО», «должность».
3 Изменить должность сотрудника «Сидоров Алексей Николаевич» на «Старший тестировщик» и повысить зарплату до 70000.
4 Очистить полностью таблицу «employee_projects».
5 Вывести все активные проекты с бюджетом более 300000.
6 Наполнить таблицу «projects» новыми проектами, используя транзакцию.
7 Вывести общее количество отработанных часов по каждому проекту. Столбцы «проект», «общее количество часов».

Схема базы данных

Таблицы связаны между собой через ключи: customers.id → orders.customer_id, products.id → order_items.product_id, orders.id → order_items.order_id, categories.id → products.category_id

customers

Таблица клиентов

id INT PRIMARY KEY
full_name VARCHAR(100)
email VARCHAR(100)
phone VARCHAR(20)
registration_date DATE

orders

Таблица заказов

id INT PRIMARY KEY
customer_id INT
order_date DATE
total_amount DECIMAL(10,2)
status VARCHAR(20)

products

Таблица товаров

id INT PRIMARY KEY
product_name VARCHAR(100)
category_id INT
price DECIMAL(10,2)
stock_quantity INT

categories

Таблица категорий товаров

id INT PRIMARY KEY
category_name VARCHAR(100)
parent_category_id INT

Связи: orders.customer_id → customers.id | products.category_id → categories.id

SQL Запросы к базе данных

1 Вывести количество заказов для каждого клиента. Столбцы «клиент», «количество заказов».
2 Вывести всех клиентов, зарегистрированных после 2023-01-01. Столбцы «ФИО», «email», «дата регистрации».
3 Изменить статус заказа с id=1001 на «доставлен» и увеличить общую сумму на 500.00.
4 Очистить полностью таблицу «categories».
5 Вывести все товары с ценой более 5000 и количеством на складе менее 10. Столбцы «товар», «цена», «количество на складе».
6 Наполнить таблицу «products» новыми товарами, используя транзакцию.
7 Вывести общую сумму заказов по каждому статусу. Столбцы «статус заказа», «общая сумма».
